### 1.1、容器查看-ps

显示本地容器列表，但默认不显示关闭的容器，只显示运行的容器，除非加上`-a`。

用法：
`docker ps [-显示所有容器，默认只显示运行中的]`

![](/assets/img/Pasted image 20251211010605.png)

### 1.2、容器详细信息-inspect

显示容器的详细信息

用法：
`docker inspect [容器名称或ID]`

![](/assets/img/Pasted image 20251211010749.png)

### 1.3、容器创建-run

创建容器

用法：
`docker run [options] 镜像名称`

后台运行容器

![](/assets/img/Pasted image 20251211011307.png)

前台运行容器
![](/assets/img/Pasted image 20251211011458.png)

```
-i 交互式创建
-t 创建一个伪终端
-d 后台执行
--name 容器名称
 /bin/bash 在伪终端中执行的命令
```


### 1.4、容器删除-rm

删除一个本地容器，默认情况下删除的容器必须是关闭状态，如果要删除一个运行中的容器，可以先关闭再删除，也可以`-f`强制删除

用法：

`docker rm [容器名称或者ID] [-f,代表强制删除，用来删除运行中的容器]`

![](/assets/img/Pasted image 20251211012128.png)

### 1.5、容器执行命令-exec

在运行容器中执行一个命令

用法：
`docker exec [容器名称或者ID] command`

![](/assets/img/Pasted image 20251211013506.png)

### 1.6、容器信息输出-attach

将一个运行容器的标准输出、错误输出、标准输入调入前台

默认容器都会在后台运行，如果想进入容器内，就可以使用该命令，这样就可以交互式的在容器里执行命令了

用法：
`docker attach [容器名称或者ID]`

![](/assets/img/Pasted image 20251211013834.png)

attach和exec的区别：

`使用attach，在exit退出容器后，容器会被关闭，而exec不会`
