**一、访问tomcat网站管理页面**

打开浏览器访问
![](/assets/img/Pasted image 20260131152548.png)

看到如上页面，该页面是tomcat的默认网站，还提供了以下功能：
- server status 查看服务器的状态，包括Linux主机的信息，tomcat版本信息，资源使用情况等
- manager app 管理网站
- host manager 虚拟主机的管理

默认情况下这三个功能需要提供账号密码，如果没有可以去修改配置文件，设置用来访问的账号密码。tomcat默认不允许远程访问管理页面，只允许localhost；默认没有任何用户；web.xml/context.xml有访问限制。

![](/assets/img/Pasted image 20260131153236.png)
配置管理用户
编辑conf/tomcat-users.xml
在文件里加上：
```
<role rolename="manager-gui"/>
<role rolename="admin-gui"/>

<user username="admin" password="admin123"
      roles="manager-gui,admin-gui"/>
```


编辑webapps/manager/META-INF/context.xml
解除默认的localhost限制，否则永远403

找到并修改（或者直接删除整个Value，我选择删掉）：
```
<Valve className="org.apache.catalina.valves.RemoteAddrValve"
       allow="127\.0\.0\.1|::1" />
```

同样修改webapps/host-manager/META-INF/context.xml

随后重启tomcat即可
```
[root@192 bin]# sh shutdown.sh
  
[root@192 bin]# sh startup.sh 

```

此时重新访问三个功能点会出现输入框，输入配置的账号密码即可

![](/assets/img/Pasted image 20260131160705.png)

manager app中主要管理的是网站是否发布的操作，start=发布网站

stop=停止发布网站，reload=重新加载，undeploy=卸载/删除网站（慎用），expire session=会话过期时间

与war文件部署相关的设置，包括定义war文件存放的位置，以及上传
war是一个可以直接运行的web模块，通常用于网站，打包部署。拿tomcat来说，把war包放在webapps目录下，然后启动tomcat，这个包就会自动解压，相当于发布了。

war包是Sun提出的一种web应用程序格式，与jar类似，是很多文件的压缩包。war包中的文件按照一定的目录结构来组织。简单来说，war包是JavaWeb程序打的包，war包里面包括写的代码编译成的class文件，依赖的包，配置文件，所有的网站页面，包括html，jsp等。一个war包可以理解为是一个web项目，里面是项目的所有东西。

![](/assets/img/Pasted image 20260131161245.png)

configuration 定义TLS(安全传输协议)配置文件

diagnostics 检查网站在启动，重新加载或卸载时，是否造成内存溢出，这个操作会触发垃圾回收机制，生产环境中慎用

TLS connector configuration diagnostics 加密诊断，可以帮助用户诊断加密是否有问题


**二、发布静态页面**

静态页面：不执行后端代码，不访问数据库，基本无交互，比如index.html

```
[root@192 webapps]# pwd
/opt/tomcat1/webapps
[root@192 webapps]# ls
docs  examples  host-manager  manager  ROOT
[root@192 webapps]# mkdir test
[root@192 webapps]# echo '<%= new java.util.Date() %>'  > test/index.jsp
```
使用浏览器访问http://ip:8080/test，每次刷新时间都会改变
