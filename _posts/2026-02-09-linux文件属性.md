
在Linux中，文件名和目录名最大长度是255个字符，而且和Windows相反，Linux区分大小写。


**一、文件类型与ugo权限**

`ls -la`命令详细查看文件类型

```
-普通文件
d目录文件
b区块文件
c字符设备文件，例如键盘鼠标等
s套接字文件
p管道文件
|链接文件

```

`file /etc/hosts`查看文件类型


什么是ugo：u代表文件所有者，g代表文件所属的用户组，o是others，即其他人，既不是文件所有者，也不在所属组内的系统其他用户。每个对象都有三种基本权限位。

补充：a(all)代表以上所有对象，即u+g+o

![](assets/img/Pasted image 20260209223732.png)

拿上图来说，图中左数第一列，每一个文件或目录都有10个字符，第一个字符代表它们的文件类型，后面9个字符以3个为一组，分别代表属主、属组、其他人的权限。r代表读，w代表写，x代表执行，这三个权限的位置不会改变，没有权限就用`-`在对应的位置表示。


Linux系统按照文件所有者、文件所有者的同组用户和其他用户来规定不同的文件访问权限，但是对于root用户来说，一般的文件权限对它不起作用。


**二、更改文件属性**

2.1、chown
修改文件所有者，也可以同时修改文件所属组

语法：
```
chown -R 属主 文件名
chown -R 属主:属组 文件名
```

-R可以看情况加，它可以递归更改文件的属性


2.2、chgrp
修改文件的属组

语法：
```
chgrp -R 属组 文件名
```

2.3、chmod
linux的权限属性有两种方法：数字或符号。

r = 4
w = 2
x = 1

如果你要给某个文件设置权限：
owner：rwx = 4 + 2 +1 = 7
group：rwx = 4 + 2 + 1 =7
others： --- = 0 + 0 + 0 = 0

语法：
```
chmod -R xyz 文件或目录
chmod -R 754 1.sh
chmod -R u=rwx,g=rx,o=r 1.sh
```

xyz即文件权限属性的值


如果要去掉所有人的可执行权限：
```
chmod a-x 1.sh
```

给1.sh添加可执行权限：
```
chmod +x 1.sh
```
