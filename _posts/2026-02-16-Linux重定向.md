**1、Linux重定向介绍**

本质：改变命令输入或输出的默认去向。

在Linux，一切基于文件描述符：

| 文件描述符 | 名称     | 说明   |
| ----- | ------ | ---- |
| 0     | stdin  | 标准输入 |
| 1     | stdout | 标准输出 |
| 2     | stderr | 标准错误 |


**2、输出重定向**

2.1、覆盖写入`>`
```
ls > list.txt
```
把ls命令的输出写入list.txt，如果文件存在会覆盖


2.2、追加写入`>>`

```
echo "hello" >> log.txt
```

hello会被加到log.txt后面，而不会覆盖原有内容


**3、错误重定向**

默认情况下，错误信息会输出到终端

如果想单独保存错误输出：
```
command 2> error.txt
```

**4、同时重定向标准输出和错误**
```
command > all.txt 2>&1
```

`2>&1`表示把错误输出`2`重定向到标准输出`1`当前指定的地方，即all.txt


**5、输入重定向**

```
wc -l < file.txt
```
把file.txt作为输入，统计有多少行内容


**6、Here Document（<<）**

```
cat << EOF
hello
rain!
hahahah
EOF
```

终端会输出：
```
hello
rain!
hahahah
```

常用于写脚本，写配置文件

写入文件：
```
cat <<EOF > output.txt
第一行内容
第二行内容
EOF
```

追加和上面同理


7、**/dev/null**

如果想执行某个命令，又不希望在终端显示输出，可以将输出重定向到/dev/null，它是一个特殊的文件，任何写入到这里的内容都会被丢弃，无法读取，将命令的输出重定向到这里，可以起到禁止输出的作用。
```
command > /dev/null
```


如果想禁止标准输出和错误输出(后台任务常用)：
```
command > /dev/null 2>&1
```

如果想分离stdout和stderr
```
command > out1.txt 2> err.txt
```
