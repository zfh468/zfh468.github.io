除了使用模板创建虚拟机，克隆也可以批量创建虚拟机

### 1、克隆分类

完整克隆：对原始虚拟机完整拷贝，删除原始虚拟机，对克隆机器不影响，需要较多的空间。


链接克隆：对原始虚拟机`硬盘`的链接，原始虚拟机删除，链接失败。




###  2、完整克隆


被克隆的机器最好先做成模板机，否则很多唯一性的东西需要手动删除


**2.1、手动克隆（理解原理用）**


a、复制一个磁盘文件
```
cp /var/lib/libvirt/images/vm1.qcow2 /var/lib/libvirt/images/vm1_clone1.qcow2
```


b、复制一个虚拟机的xml文件
```
virsh dumpxml --domain vm1 > /etc/libvirt/qemu/vm1_clone1.xml
```


c、修改xml文件，将原始机器的唯一性配置删除

修改虚拟机名字
删除uuid、mac地址（直接删除整行）
修改磁盘路径
如果需要，修改graphics标签的vnc端口为其他端口，避免与5980冲突


d、根据xml文件导入虚拟机
```
virsh define --file /etc/libvirt/qemu/vm1_clone1.xml
```


e、克隆完成，检查

```
# 启动新机
virsh start vm1_clone1

# 检查列表
virsh list --all
```

**2.2自动克隆**

简单，不容易出错，自动完成磁盘复制、生成新的mac地址、更新UUID

自动克隆前，需要关停被克隆机子
```
virsh shutdown vm1
```

执行命令
```
virt-clone --original vm1 --name vm_clone_01 --file /var/lib/libvirt/images/vm_clone_01.qcow2
```

启动虚拟机
```
virsh start vm_clone_01
```



### 3、链接克隆


a、创建一个链接克隆磁盘，必须是qcow2格式

```
qemu-img create -f qcow2 \
-b /var/lib/libvirt/images/vm1.qcow2 \
-F qcow2 \
/var/lib/libvirt/images/vm_link_01.qcow2
```

-b指定母机镜像


```
[root@192 ~]# ll -h /var/lib/libvirt/images/vm_link_01.qcow2
-rw-r--r-- 1 root root 193K Mar 10 13:18 /var/lib/libvirt/images/vm_link_01.qcow2

# 大小仅为193k的链接磁盘
```

查看是否链接到母机
```
qemu-img info /var/lib/libvirt/images/vm_link_01.qcow2
```


b、创建链接克隆虚拟机
```
virt-install \
--name vm_link_01 \
--memory 2048 \
--vcpus 2 \
--disk path=/var/lib/libvirt/images/vm_link_01.qcow2,bus=virtio \
--network network=default,model=virtio \
--graphics vnc,listen=0.0.0.0,port=5990,password=123456 \
--os-variant rhel8.0 \
--import \
--noautoconsole
```

即使vm_link_01被删掉各种文件，也不影响母机vm1,只要删掉vm_link_01.qcow2，然后重复以上链接克隆步骤，几秒内就能恢复一个全新的系统
